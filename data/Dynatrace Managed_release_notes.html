New features and enhancements

Cluster resilience improvements

- To prevent a data loss in case of a severely undersized disk for Transaction and Session Replay storage, the cluster node will disable OneAgent traffic on itself and will no longer accept incoming data. For example, this might be a result of a disk size misconfiguration. At startup, the cluster node won't be able to start successfully. You'll receive a dedicated cluster event when such a situation happens.

- To improve the resilience of cluster nodes replication between DCs, a cluster node will not start if data synchronization hasn't been finished. The procedure of data replication for Premium High Availability deployements has been updated.
